# Density-First Core Preference Identification System

## A Stable Preference Inference Pipeline from Noisy Behavioral Observations

---

## 1. Problem Definition (formal and precise)

### Goal

Given a set of noisy behavioral observations extracted from user interactions, infer the user’s **stable core preferences** such that:

* Only **well-supported, semantically coherent preferences** are identified
* Rare but plausible preferences are **not discarded**
* The system avoids **heuristic thresholds, pre-filtering, and arbitrary scoring**
* The output is suitable for **LLM conditioning and personalization**

### Correct framing (critical)

> We are **not detecting behaviors**.
> We are **estimating the high-support regions of a user’s latent preference distribution** in semantic embedding space.

This framing aligns the system with:

* Density estimation
* Support recovery
* Preference modeling
* User representation learning

**Key references**

* Zhang et al., *Deep Learning for User Interest Modeling*, SIGIR 2019
* Wu et al., *User Embedding for Recommendation Systems*, WWW 2020
* Wang et al., *Representation Learning for User Preferences*, ACM TOIS 2021

## 2. System Inputs

### 2.1 Behavioral Observation Schema

Each observation is treated as a **noisy sample** from the user’s latent preference distribution.

```json
{
  "observation_id": "obs_001",
  "behavior_text": "Prefers visual explanations",
  "embedding": [1536 floats],
  "credibility": 0.87,
  "timestamp": 1704326400
}
```

### 2.2 Explicit Assumptions (must be stated)

1. **Embeddings**

   * Generated by a sentence-level embedding model
   * Semantic similarity is best captured by cosine similarity

2. **Credibility**

   * Scalar ∈ [0, 1]
   * Represents posterior belief that this observation reflects a true preference signal
   * Computed upstream using factors such as:

     * reinforcement
     * recency
     * clarity
     * user confirmation

> This system does **not** recompute credibility.
> Credibility is treated as an **external belief signal**.

This decoupling is essential for modularity and correctness.

**References**

* Steck, *Implicit Feedback Confidence*, RecSys 2019
* Liang et al., *Modeling Confidence in Implicit Feedback*, RecSys 2018

---

## 3. Mathematical Interpretation of Inputs

Let:

* ( x_i ) be the embedding of observation ( i )
* ( w_i ) be its credibility

We interpret the data as a **weighted empirical distribution**:

[
\hat{f}*u(x) = \sum*{i=1}^{N} w_i , \delta(x - x_i)
]

### Interpretation

* Each observation contributes **mass** proportional to its credibility
* High-credibility observations increase local density
* This is standard weighted density estimation

This is not heuristic; it is textbook statistics.

**References**

* Silverman, *Density Estimation for Statistics and Data Analysis* (foundation)
* Chen et al., *Weighted Kernel Density Estimation for Uncertain Data*, IEEE TKDE 2019
* Kpotufe & Garg, *Adaptive Density Estimation*, COLT 2020

## 4. Preprocessing (minimal, mandatory, justified)

### 4.1 Embedding Normalization

```text
x_i ← x_i / ||x_i||
```

### Why this is required

* Sentence embeddings encode meaning via **direction**
* Euclidean distance on normalized vectors ≡ cosine distance
* Density-based clustering assumes metric consistency

Skipping this breaks density estimation.

**References**

* Reimers & Gurevych, *Sentence-BERT*, EMNLP 2019
* Ethayarajh, *How Contextual Are Contextualized Embeddings?*, EMNLP 2019

---

### 4.2 What is intentionally NOT done

* ❌ No pre-filtering
* ❌ No anomaly detection
* ❌ No threshold-based pruning
* ❌ No minimum-N rejection

**Reason:**
In density theory, *noise is defined relative to density*, not prior to it.

**References**

* Ester et al., *DBSCAN*, KDD 1996
* Schubert et al., *On the Notion of Noise in Clustering*, ACM TKDD 2018
* Campos et al., *Evaluation of Unsupervised Outlier Detection*, DMKD 2019

## 5. Core Inference: Density-Based Clustering

### 5.1 Why density-based clustering

Preferences are:

* Multi-modal
* Non-spherical
* Unknown in number
* Embedded in noise

Density-based clustering directly models these properties.

### 5.2 Algorithm Choice: HDBSCAN

HDBSCAN is selected because it:

* Estimates density-connected components
* Models noise explicitly
* Produces **cluster stability**
* Approximates density level-set recovery

**References**

* Campello et al., *Hierarchical Density Estimates for Clustering*, PAKDD 2015
* McInnes et al., *HDBSCAN*, JOSS 2017
* Schubert et al., *DBSCAN Revisited*, ACM TKDD 2019

---

### 5.3 Configuration (adaptive, defensible)

```text
min_cluster_size = max(2, floor(log(N)))
metric = euclidean
cluster_selection_method = "eom"
```

#### Justification

* Single observations cannot define preferences
* Parameter scales with data volume
* No epsilon or fixed density thresholds
* Avoids arbitrary constants

---

### 5.4 Weight handling

Credibility is treated as **sample mass** in density estimation.

Implementation options (must be documented):

* True weighted clustering (if library supports it correctly)
* Equivalent approximation (sample duplication / post-hoc weighting)

Transparency here is essential for reviewer trust.

* Optional membership probabilities

## 7. Epistemic Interpretation (critical layer)

This system distinguishes **epistemic states**, not just cluster labels.

### Why this matters

Most systems incorrectly equate:

* “not dense” = “noise”

That is false.

---

## 8. Epistemic States (formal definitions)

### 8.1 CORE (stable preference)

A cluster is CORE if:

```text
cluster_stability ≥ median(cluster_stabilities)
```

Properties:

* Relative, not absolute
* User-specific
* Non-parametric
* Robust across sparse and dense users

These clusters define **core preferences**.

**References**

* Chaudhuri & Dasgupta, *Rates of Convergence for the Cluster Tree*, NeurIPS
* von Luxburg, *Clustering Stability*, Foundations & Trends in ML

---

### 8.2 INSUFFICIENT_EVIDENCE (plausible but unsupported)

Applies to:

* High-credibility singletons
* Small or unstable clusters
* Emerging or infrequent behaviors

Formally (example):

```text
credibility ≥ upper_quantile
AND
not CORE
```

Interpretation:

* Plausible preference hypothesis
* Retained for future reinforcement
* Not surfaced to LLMs

This resolves cold-start and rare-preference issues **without violating density theory**.

**References**

* Rashid et al., *Learning New User Preferences*, IUI
* Zhao et al., *Cold-Start Recommendation via Meta-Learning*, NeurIPS 2020
* Liu et al., *Uncertainty-Aware User Modeling*, WWW 2021

---

### 8.3 NOISE (discardable)

Observations that are:

* Low credibility
* Isolated
* Unstable relative to density structure

Only these are safely ignored.

---

## 9. Preference Confidence Estimation

For each CORE cluster:

```text
confidence = normalized_cluster_stability
```

Optional:

* Multiply by temporal coverage (explicitly documented)

### Why this is correct

* Stability measures persistence across density levels
* This directly corresponds to preference robustness

No composite heuristics.
No bonuses.
No penalties.

**References**

* Ben-David et al., *Stability and Learnability*, JMLR
* Ackerman et al., *Cluster Quality Measures*, JMLR 2020

---

## 10. Semantic Labeling (LLM-assisted, inference-free)

### Role of LLMs

LLMs are used **only** for:

* Semantic summarization
* Human-readable labeling

They do **not** affect:

* Core detection
* Confidence
* Inclusion or exclusion

### Input

* Medoid texts of cluster
* Optional centroid keywords

### Output

```text
"Visual Learning Preference"
```

This is presentation, not modeling.

**References**

* Ribeiro et al., *Why Should I Trust You?*, KDD
* Jacovi & Goldberg, *Faithful Model Interpretation*, ACL 2020
* Gao et al., *LLMs as Semantic Compressors*, arXiv 2023

## 12. What is explicitly excluded (by design)

* Pre-clustering anomaly detection
* Tiering (PRIMARY / SECONDARY)
* Hard thresholds
* Minimum-N rules
* Temporal decay hacks
* Reinforcement curves
* Composite scoring

This is **discipline**, not simplicity.
